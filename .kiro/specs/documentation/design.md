# Documentation - Design

## Structure

### Documentation Hierarchy
```
docs/
├── ARCHITECTURE.md      # System design
├── DEPLOYMENT.md        # Deployment guide
├── TROUBLESHOOTING.md   # Problem solving
├── ONBOARDING.md        # New developer guide
├── ENVIRONMENTS.md      # Environment config
├── SLO.md              # Service objectives
├── API.md              # API reference
└── DESIGN-SYSTEM.md    # UI/UX guide
```

### Content Strategy
- Start with overview
- Provide examples
- Include troubleshooting
- Link related docs
- Keep updated

### Writing Style
- Active voice
- Short paragraphs
- Bullet points
- Code blocks
- Visual aids

## Templates
Each doc follows consistent structure:
1. Overview
2. Quick start
3. Detailed sections
4. Examples
5. Troubleshooting
6. Related docs
